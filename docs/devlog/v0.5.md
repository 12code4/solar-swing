# devlog: v0.5, or the time I made two AIs work a job site

Solar Swing v0.5 is done and I barely wrote any code this round. That was the experiment.

Quick recap for anyone new: Solar Swing is my browser game where you jetpack around a
scaled-down solar system with real orbital mechanics. No ship, no autopilot. Just you,
a thruster that overheats, and gravity.

For this version I tried splitting the work between two AI models. One acted as the
planner. We argued about scope, wrote a spec with acceptance criteria, and locked in
rules like "craters must live in the same function that drives collision, or players
will float above the terrain." Then a second model executed the spec cold, no
conversation history, just the documents. Foreman and builder, basically.

It worked better than I expected. The builder shipped all five spec items: five octaves
of terrain noise, actual crater bowls on Mercury and Pluto, a fresnel shader that makes
atmospheres glow at the horizon, bloom on desktop, and the feature I care most about,
a trajectory line that turns green when you achieve orbit. It even caught a rendering
bug I did not know existed. The post-processing pipeline silently drops tone mapping
unless you add an output pass, so desktop and mobile would have looked different forever.

The honest tradeoff: writing a spec that precise took real time. For a small change it
would be overkill. For five features touching physics, shaders, and rendering, it paid
for itself, because nothing came back wrong.

Best number this version: the prediction line now looks minutes ahead in deep space
instead of 32 seconds. You can see a whole orbit before you commit to it.

Next up: playtesting whether escaping Earth at 45g feels like a skill or a wall.
