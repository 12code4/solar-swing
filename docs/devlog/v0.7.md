# v0.7 - refactor

No new features this time. On purpose. This one was about turning the game from a single 1810-line HTML file into a real Vite and TypeScript project with proper modules.

I put this off for a while because it felt like busywork. The single file was fast to iterate on and I liked that. But it was getting hard to hold in my head, and every time I opened it I had to scroll past the whole solar system to find the two lines I wanted. So the job for v0.7 was simple to state and easy to get wrong: split it up and change nothing else.

That "change nothing else" part was the whole game. A refactor that also fixes a bug is how you end up with a silent new bug. So the rule was zero observable change. Same physics, same feel, same visuals, same bugs. Yes, even the Jupiter one. That stays a separate job.

The safety net was a git tag. Before touching anything I tagged the old build as v0.6, so I always have the exact thing to compare against. Then I pulled the code apart leaf first, the pure maths first, the main loop last, and ran the type checker after every single extraction instead of at the end. When something broke I knew it was the last move, not a mystery from an hour ago.

The one real decision was how to share state. The clean answer is to thread a context object through everything. The safe answer, for a refactor measured on sameness, is grouped singletons: the same mutable objects the old file used, just living in modules now. I went with safe. The pretty version is written down in the backlog for when I have tests to catch a regression.

It splits into about eighteen files now. The three things that had to stay true still hold: one shape function drives both the terrain you see and the ground you hit, one gravity function feeds both the sim and the prediction line, and all the tuning lives in one CONFIG. Same game. Now I can find things.

One small extra, in its own commit after I checked parity: I bumped the planet texture URLs from 2k to 4k. Free sharpness. The proper fix for close-up blur is detail levels, and that is in the backlog too.
