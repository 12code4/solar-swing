# v0.8 - salvage run

The game finally has a reason to go anywhere. Until now Solar Swing was a physics toy: fun to orbit, nothing to want. v0.8 adds a roguelite layer on top and, honestly, it was overdue.

The loop is simple. About 130 salvage shards are scattered through the system with a seeded random, so everyone gets the same layout. Rocky planets get shards on the surface and a ring of them in orbit. Gas giants get orbital rings plus one fat shard worth 25 sitting on the core, past the NO RETURN line. You can grab it, but you are not flying home afterwards. That one is a paid dive and I love it.

Death matters now. The sun kills you, drifting into the void kills you, and landing harder than 30 m/s kills you. That last one changes everything, because aerobraking and matching a planet's velocity used to be optional style points. Now they are survival. Die and the run ends, but salvage banks anyway, and you spend it between runs on permanent upgrades. Better cooling, tougher hull, more thrust. The thrust upgrade caps at 44.1 on purpose. Big planet surface gravity is 45, and the rule that you cannot hover your way off Jupiter is the soul of the game. No upgrade gets to break it.

First landing on each body also pays a discovery bonus, so there is a checklist itch: 10 entries, Mercury to Pluto plus the belt.

The other half of this update is a debug menu on the tilde key. Live stats, a time scale, and an auto-generated editor for every number in CONFIG. Because the prediction line reads the same CONFIG as the live sim, edits show up in both instantly. I built it to tune the shard economy and ended up tuning everything with it. Should have built it three versions ago.

Tradeoffs: meta progression needs saving, so this breaks my own no-save-load rule with a small localStorage blob. Sandbox saving is still backlogged. And the debug menu is desktop only for now.
